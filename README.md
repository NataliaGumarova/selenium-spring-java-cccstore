## 1. Установка JDK
Java Development Kit (JDK) — это набор инструментов, необходимых для разработки и выполнения программ на языке Java. Он включает в себя компилятор, виртуальную машину (JVM) и библиотеки, которые обеспечивают создание, компиляцию и запуск Java-приложений. Установка JDK необходима для того, чтобы разработчик мог писать и тестировать код на Java.
Современным и рекомендуемым источником для скачивания JDK является проект `Eclipse Adoptium`, предоставляющий сборки под названием `Temurin`. Этот дистрибутив является бесплатным, открытым и сертифицированным по стандарту Java SE, что гарантирует совместимость и качество. В отличие от Oracle JDK, Temurin свободно доступен для любых целей, включая коммерческое использование, и поддерживается сообществом и Eclipse Foundation.
Для загрузки JDK с сайта Adoptium (https://adoptium.net/temurin/releases) следует выбрать необходимую версию Java, пакет в зависимости от операционной системы и тип JDK, после чего скачать и установить дистрибутив согласно инструкциям для вашей платформы. После установки рекомендуется настроить переменные среды, такие как PATH и JAVA_HOME, чтобы операционная система и инструменты разработки могли корректно обнаружить JDK.
Использование Adoptium Temurin JDK обеспечивает разработчикам высококачественные и надежные средства разработки Java-приложений с регулярными обновлениями и поддержкой различных операционных систем. Это делает данный дистрибутив удобным и современным выбором для программирования на Java.

## 2. Installing IntelliJ IDEA
Процесс установки IntelliJ IDEA включает несколько ключевых шагов, которые обеспечивают корректное и эффективное использование данного интегрированного средства разработки. Важно отметить, что для успешного скачивания и активации программы рекомендуется использовать VPN, поскольку доступ к официальному сайту JetBrains из некоторых регионов может быть ограничен. Использование VPN не только обеспечивает доступ к официальному ресурсу, но и позволяет избежать задержек и перебоев в соединении, что критически важно при скачивании крупных установочных файлов и обновлений.

Первым шагом является переход на официальный сайт JetBrains, где предлагается выбрать подходящую версию IntelliJ IDEA. Особое внимание стоит уделить Ultimate-версии, которая обладает расширенным функционалом, необходимым для профессиональной разработки, включая поддержку множества языков программирования и интеграцию с различными фреймворками и инструментами. Ultimate-версия значительно расширяет возможности по сравнению с бесплатной Community-версией, что делает её предпочтительным выбором для выполнения сложных и масштабных проектов.

Далее следует скачивание установочного файла и запуск инсталлятора. Во время установки рекомендуется внимательно следовать инструкциям, выбирая необходимые компоненты и настройки, чтобы адаптировать среду под требования конкретного проекта. После завершения установки необходимо выполнить активацию продукта с помощью лицензионного ключа, который можно получить либо через официальный канал JetBrains, либо приобретя подписку.


 ## 3. Loading the Selenium Java Client Driver
 Подготовка к использованию Selenium WebDriver с Java начинается с загрузки Selenium Java Client Driver. Этот драйвер представляет собой набор библиотек, необходимых для интеграции автоматизации браузера в Java-проекты.

Для начала следует перейти на официальный сайт Selenium и скачать последнюю версию Selenium Java Client Driver. После загрузки архивного файла его нужно распаковать, внутри которого находится несколько JAR-файлов. Эти файлы необходимо импортировать в среду разработки, например, Eclipse, добавив их в путь сборки проекта (Java Build Path). Для этого в настройках проекта выбирается вкладка "Libraries", и с помощью функции "Add External JARs..." указываются все соответствующие JAR-файлы из распакованной папки.

Этот процесс обеспечивает подключение всех необходимых библиотек Selenium к проекту и позволяет начать написание автотестов на Java с использованием WebDriver. Важно выбирать актуальную версию драйвера для совместимости с браузерами и Java.
 

 ## 4. Browser driver installation: Windows
 Процесс установки драйвера браузера для Selenium WebDriver на Windows включает несколько важных шагов для правильной настройки и успешного запуска автоматизированных тестов.

В первую очередь, необходимо определить, какой именно браузер будет использоваться для тестирования, так как для каждого браузера нужен свой драйвер: для Google Chrome — ChromeDriver, для Mozilla Firefox — GeckoDriver, для Microsoft Edge — EdgeDriver. Это связано с тем, что каждый драйвер обеспечивает совместимость Selenium с конкретной версией браузера.

Далее следует скачать соответствующий драйвер с официального сайта или репозитория. Например, ChromeDriver можно загрузить с официальной страницы проекта, где нужно выбрать версию, совпадающую с установленной версией браузера Chrome. Узнать версию браузера можно, зайдя в меню «Справка» или введя в адресной строке chrome://settings/help. Полученный архив с драйвером нужно распаковать.

Для удобства запуска драйвера рекомендуется добавить путь к распакованному файлу chromedriver.exe (или аналогичному для других браузеров) в системную переменную PATH Windows. Это позволит системе находить драйвер автоматически при запуске тестов. Добавление в PATH делается через «Панель управления» → «Система» → «Дополнительные параметры системы» → «Переменные среды». В разделе системных переменных выбирается PATH, и в него добавляется полный путь к папке с драйвером.

Для проверки правильности установки можно открыть командную строку и ввести команду пола вашего драйвера, например,
```bash
chromedriver --version
```
 Если драйвер установлен корректно, появится информация о версии драйвера.

## 5. Selenium Server
(Standalone)
Selenium Server Standalone — это Java-базированный сервер, который играет роль посредника между тестовыми скриптами Selenium WebDriver и веб-браузерами. Он позволяет запускать тесты удалённо на разных машинах и управлять несколькими сессиями браузеров одновременно. Это особенно полезно при использовании Selenium Grid, где сервер выступает в роли центрального узла (хаба) или узла-исполнителя (ноды) для распределённого и параллельного тестирования.

Данный сервер помещён в один исполняемый JAR-файл, который содержит все необходимые компоненты Selenium WebDriver, что обеспечивает удобство установки и запуска без необходимости отдельно загружать клиентские библиотеки. Запустить сервер можно командой в терминале:
```bash
java -jar selenium-server-standalone-<версия>.jar
```
После запуска сервер принимает команды от тестов и управляет браузерными сессиями.

Основные преимущества Selenium Server Standalone заключаются в возможности управлять браузерными тестами на удалённых или нескольких устройствах, обеспечении кроссбраузерности, а также поддержке интеграций с такими фреймворками, как TestNG и JUnit. Его удобно применять для небольших и средних проектов, а также при подготовке к масштабированию тестовой инфраструктуры.

Использование Selenium Server Standalone позволяет с минимальными усилиями настроить удалённое выполнение автоматизированных тестов и обеспечивает совместимость с различными языками программирования и браузерами, что делает его важным компонентом в экосистеме автоматизированного тестирования.

## 1.1.1 Connect Selenium to IntelliJ project
Для подключения Selenium к проекту в IntelliJ IDEA необходимо выполнить несколько последовательных шагов.

Сначала следует скачать Selenium Java Client Driver с официального сайта Selenium (www.selenium.dev/downloads). После загрузки архива его распаковывают на локальном диске. Далее открывают проект в IntelliJ IDEA и переходят в меню File → Project Structure. В появившемся окне выбирают раздел Modules, затем нажимают на кнопку "+" (добавить) и выбирают пункт JARs or directories…. В открывшемся диалоговом окне указывают путь к папке с распакованными файлами Selenium и выделяют все JAR-файлы как из корневой папки, так и из папки libs. После этого подтверждают выбор кнопкой OK, что добавляет необходимые библиотеки в класс-путь проекта.

Этот способ позволяет вручную подключить Selenium библиотеки к проекту, обеспечивая доступ к функционалу WebDriver для автоматизации браузерных тестов.
